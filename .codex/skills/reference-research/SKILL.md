# Reference Research

Use this skill when a task depends on outside examples or current patterns.

## Trigger

Use when:
- the user names another app, product, article, or design reference
- the task asks for research, examples, comparisons, or latest patterns
- implementation quality depends on external precedent

## Workflow

1. Confirm the exact target product or source.
2. Gather only sources that change planning or design decisions.
3. Separate observed facts from your inference.
4. Extract patterns worth copying and anti-patterns worth avoiding.
5. End with a recommendation that changes next actions.

## Output Format

- Key findings
- Relevant patterns
- Risks
- Open questions
- Recommendation

## Rules

- Prefer primary or directly relevant sources.
- If the exact product is ambiguous, say so before concluding.
- Do not dump raw links without synthesis.
- Do not turn research into implementation planning unless asked.

## Fail Conditions

- No source grounding
- Listing examples without a conclusion
- Mixing fact and inference without labeling them
